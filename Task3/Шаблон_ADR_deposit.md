### <a name="_qmphm5d6rvi3"></a>**Решение**

#### Диаграммы

Для визуализации архитектуры используются диаграммы модели C4:

1. [diagram_context.puml](diagram_context_deposit.puml) — Диаграмма контекста (уровень C4-1): показывает основные внешние системы и их взаимодействие.
2. [diagram_containers.puml](diagram_containers_deposit.puml) — Диаграмма контейнеров (уровень C4-2): детализирует системы банка.

#### Логика принятия решений

1. **Отказ от прямых вызовов АБС**  
   - АБС перегружена и масштабируется только вертикально.  
   - Решение: введение **Kafka** как брокера событий. Заявки публикуются в топик `deposits`, АБС потребляет их асинхронно.

2. **Управление ставками в АБС**  
   - Сейчас ставки — в XLS. Это антипаттерн.  
   - Решение: создание таблицы `deposit_rates` в АБС. Данные реплицируются в кэш (Redis) для быстрого доступа сайта и интернет-банка.

3. **Горизонтальное масштабирование интернет-банка**  
   - Сейчас работает в одном ЦОД.  
   - Решение: развёртывание в двух ЦОД, балансировка нагрузки (NGINX), DNS-фаиливер. Использование stateless-сервисов.

4. **Подтверждение операций**  
   - СМС-механизм — в ядре.  
   - Решение: вызов существующего API СМС-подтверждения из интернет-банка. Не требует доработки ядра.

5. **Безопасность**  
   - Передача ФИО и телефона — чувствительные данные.  
   - Решение: TLS 1.2+, валидация на стороне сервера, логирование без данных.

6. **MVP-подход**  
   - Пока не нужна микросервисная архитектура всего интернет-банка.  
   - Решение: добавить **новый микросервис** `DepositService` внутри интернет-банка, который будет работать с Kafka.

### <a name="_bjrr7veeh80c"></a>**Альтернативы**

| Альтернатива | Описание | Почему не выбрана |
|-------------|--------|------------------|
| **Прямые вызовы API АБС (REST/SOAP)** | Упрощённая интеграция без очередей | Высокий риск перегрузки АБС. Нарушает требование +R01. |
| **Собственный шлюз для СМС** | Создать новый шлюз вместо использования существующего | Увеличивает сложность. Есть рабочее ядро — лучше его использовать. |
| **Хранение ставок в отдельной системе (например, Config Service)** | Вынести ставки за пределы АБС | Усложняет согласование. АБС — естественное место для финансовых параметров. |
| **Полный рефакторинг интернет-банка на микросервисы** | Полный переход на микросервисную архитектуру | Не соответствует MVP. Слишком дорого и долго. |

#### **Недостатки, ограничения, риски**

| Риск | Описание | Митигация |
|-----|--------|---------|
| **Несовместимость интернет-банка с Kafka** | Текущая версия не поддерживает Kafka | Временное решение: адаптер (REST -> Kafka Producer). |
| **Задержка открытия депозита** | Асинхронная обработка может вызвать задержку | Уведомления клиенту по СМС помогают снизить тревожность. SLA обработки — 1 час. |
